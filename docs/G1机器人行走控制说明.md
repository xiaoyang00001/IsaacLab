# G1 机器人行走控制说明

## 当前状态

文件 [locomanipulation_g1_env_cfg.py](../source/isaaclab_tasks/isaaclab_tasks/manager_based/locomanipulation/pick_place/locomanipulation_g1_env_cfg.py) 第 61–63 行：

```python
FIXED_G1_29DOF_CFG = G1_29DOF_CFG.copy()
FIXED_G1_29DOF_CFG.spawn.articulation_props.fix_root_link = True
FIXED_G1_29DOF_CFG.spawn.rigid_props.disable_gravity = True
```

机器人的根关节被**固定**在原地（`fix_root_link = True`），重力也被禁用，因此机器人当前无法行走。

---

## 行走控制的两种方案

### 方案一：直接平移根节点（滑步，无腿部动画）

这是 [mdp/actions.py](../source/isaaclab_tasks/isaaclab_tasks/manager_based/locomanipulation/pick_place/mdp/actions.py) 中 `AgileBasedLowerBodyAction` 的**当前实际行为**。

`_apply_absolute_root_pose()`（第 124–176 行）通过直接写入根节点位姿来移动机器人，命令格式为：

```
actions = [vx, vy, wz, hip_height]
```

| 参数 | 含义 |
|------|------|
| `vx` | 前后速度 |
| `vy` | 左右速度 |
| `wz` | 转向角速度 |
| `hip_height` | 髋关节高度（默认 0.72 m） |

**效果**：机器人会平移/转向，但腿部不产生行走动画（"滑步"效果）。

**启用步骤**：
1. 解除根关节固定（见下方）
2. 在 `ActionsCfg` 中添加 `AgileBasedLowerBodyActionCfg`（无需策略文件）

---

### 方案二：完整物理行走（需要预训练行走策略模型）

使用 `AgileBasedLowerBodyAction` + ONNX/TorchScript 行走策略，腿部会产生真实的行走动画。

#### 步骤 1：解除根关节固定

修改 [locomanipulation_g1_env_cfg.py](../source/isaaclab_tasks/isaaclab_tasks/manager_based/locomanipulation/pick_place/locomanipulation_g1_env_cfg.py) 第 61–63 行：

```python
# 改为（去掉固定和禁用重力）
FIXED_G1_29DOF_CFG = G1_29DOF_CFG.copy()
# 删除以下两行：
# FIXED_G1_29DOF_CFG.spawn.articulation_props.fix_root_link = True
# FIXED_G1_29DOF_CFG.spawn.rigid_props.disable_gravity = True
```

#### 步骤 2：在 ActionsCfg 中启用下肢动作

在 `ActionsCfg` 类中添加（[locomanipulation_g1_env_cfg.py 第 225 行](../source/isaaclab_tasks/isaaclab_tasks/manager_based/locomanipulation/pick_place/locomanipulation_g1_env_cfg.py#L225)附近）：

```python
lower_body = AgileBasedLowerBodyActionCfg(
    asset_name="robot",
    joint_names=[
        "left_hip_pitch_joint",
        "left_hip_roll_joint",
        "left_hip_yaw_joint",
        "left_knee_joint",
        "left_ankle_pitch_joint",
        "left_ankle_roll_joint",
        "right_hip_pitch_joint",
        "right_hip_roll_joint",
        "right_hip_yaw_joint",
        "right_knee_joint",
        "right_ankle_pitch_joint",
        "right_ankle_roll_joint",
    ],
    obs_group_name="lower_body_policy",
    policy_path="<ONNX或TorchScript行走策略文件路径>",
)
```

#### 步骤 3：配置行走命令来源（选其一）

代码中已有两个 retargeter 实现，选择适合的输入方式：

**选项 A：VR 头显控制（头部平移/转向 → 行走命令）**

在 `teleop_devices` 中的 `handtracking` 设备里添加：

```python
from isaaclab.devices.openxr.retargeters.humanoid.unitree.g1_lower_body_standing import (
    G1LowerBodyStandingRetargeterCfg,
)

G1LowerBodyStandingRetargeterCfg(
    hip_height=0.72,
    sim_device=self.sim.device,
)
```

**选项 B：手柄摇杆控制**

```python
from isaaclab.devices.openxr.retargeters.humanoid.unitree.g1_motion_controller_locomotion import (
    G1LowerBodyStandingMotionControllerRetargeterCfg,
)

G1LowerBodyStandingMotionControllerRetargeterCfg(
    hip_height=0.72,
    sim_device=self.sim.device,
)
```

---

## 核心瓶颈

完整物理行走方案的前提是**预训练行走策略模型**（`.onnx` 或 `.pt` 文件）。`AgileBasedLowerBodyActionCfg` 中的 `policy_path` 当前为 `MISSING`，必须填入实际路径。

如果没有策略模型，只能使用**方案一（滑步）**。
