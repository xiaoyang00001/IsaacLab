## 方案一：直接平移根节点（滑步，无腿部动画）

这是 [mdp/actions.py](../source/isaaclab_tasks/isaaclab_tasks/manager_based/locomanipulation/pick_place/mdp/actions.py) 中 `AgileBasedLowerBodyAction` 的**当前实际行为**。

`_apply_absolute_root_pose()` 通过 `write_root_state_to_sim` 直接写入根节点位姿，命令格式为：

```
actions = [vx, vy, wz, hip_height]
```

| 参数 | 含义 |
|------|------|
| `vx` | 前后速度 |
| `vy` | 左右速度 |
| `wz` | 转向角速度 |
| `hip_height` | 髋关节高度（默认 0.72 m） |

**效果**：根节点平移/转向，但腿部维持站立姿（"滑步"）。

## 方案三：模拟全身骨骼数据驱动（walker_robot — 当前已启用）

第三个机器人 **`walker_robot`** 已加入场景，点击 Play 后会通过物理引擎自然行走，并由内部生成的"骨骼姿态"驱动全身关节做协调动作（步态 + 摆臂 + 腰扭 + 放松握手）。

### 开发历史

该功能经历了以下迭代优化：

1. **v1 - 基础全身行走**（commit 63fd40e）
   - 实现基础的全身骨骼数据驱动行走
   - 腿部 CPG 步态 + 手臂摆动 + 腰部扭转

2. **v2 - 自然感优化**（commit 1ae857a + e8fe4c15）
   - 添加腰部 roll 侧倾（重心转移）
   - 添加髋部 yaw 旋转（骨盆旋转）
   - 修正膝关节相位（mid-swing 弯曲）
   - 添加躯干竖向起伏（body bob）

3. **v3 - 物理驱动**（commit 93b8e66，当前版本）
   - 从动画驱动改为物理驱动
   - 解除根节点固定，启用重力
   - 移除根节点强制平移
   - 实现真实的脚地接触

### 行走机制：物理驱动 + 关节动画

当前实现采用**物理驱动**方式，实现真实行走：

1. **根节点自由运动**
   - 根节点未固定（`fix_root_link = False`）
   - 启用重力（`disable_gravity = False`）
   - 机器人通过脚与地面的接触力自然前进

2. **关节动画驱动步态**
   - 腿部做步态动作（髋摆、膝弯、踝补偿）
   - 手臂反向摆动
   - 腰部扭转和侧倾
   - 髋部旋转

**特点**：
- ✅ 真实物理行走，脚与地面产生接触力
- ✅ 动画流畅，全身协调
- ✅ 自然的重心转移和平衡
- ⚠️ 需要调整参数以保持稳定

**设计目标**：展示全身协调的物理行走，模拟骨骼捕捉数据重定向驱动真实机器人的效果。

### 概念上的数据流

```text
walking_phase(t)
   ↓
_sample_skeleton_pose(phase)         ← 等价于 retarget(mocap_data, robot_skeleton)
   ↓
全身关节目标角度（dict）
   ↓
set_joint_position_target(...)
```

`_sample_skeleton_pose` 用解析公式合成与 walking phase 同步的全身姿态，模拟一个本地 mocap 流。**结构与真实 mocap 重定向函数一致**，后续可替换为读取 ZMQ/BVH/SMPL 数据流。

### 驱动覆盖范围

| 部位 | 关节数 | 行为 |
|------|--------|------|
| 腿部 | 12 | 左右 180° 反相 CPG 步态：髋俯仰摆腿、膝弯曲（仅摆动相）、踝反向补偿 |
| 手臂 | 14 | **反向摆臂**：肩 pitch 与同侧腿前后位置反相；肘随摆动微弯 |
| 腰部 | 3 | **yaw 扭转**：小幅反向扭转，增加自然感<br>**roll 侧倾**：模拟重心转移，支撑腿侧身体倾斜 |
| 髋部 | 2 | **yaw 旋转**：左右髋关节协同旋转，产生骨盆旋转效果 |
| 手部 | ≤14 | 恒定轻微卷曲（放松握拳）；不存在则自动跳过 |

### 自然感优化

为消除"幽灵感"，实现了以下关键改进：

1. **膝关节相位修正**
   - 膝关节在腿前摆中段（mid-swing）弯曲最大
   - 符合真实人类行走的运动规律

2. **身体侧向倾摆**
   - 左腿支撑时身体向左倾，右腿支撑时向右倾
   - 模拟重心转移

3. **骨盆旋转**
   - 髋部 yaw 与腰部协同旋转
   - 增强行走的自然感

4. **物理驱动**
   - 脚与地面产生真实接触力
   - 自然的重心转移和平衡

### 摆臂相位约定

```
θ_L = phase            (左腿相位)
θ_R = phase + π        (右腿相位，180° 偏移)

左腿前后:  hip_pitch_L = default + A_hip * sin(θ_L)
左肩前后:  shoulder_pitch_L = default + A_arm * sin(θ_L)   ← 与同侧腿"位置"反相
右腿/右肩类似（用 θ_R）。
```

物理上：左腿向前迈进时，**右臂前摆**（保持角动量守恒）—这是人类行走的天然规律。

### 后续扩展方向

将 `_sample_skeleton_pose` 替换为真实数据源：

| 数据源 | 替换方式 |
|--------|----------|
| ZMQ mocap 流 | 仿照 `G1TriHandUpperBodyZeroMqRetargeterCfg`，在 `__init__` 启动 SUB，每步从队列取最新 pose |
| BVH/FBX 文件 | `__init__` 加载文件 → 在 `_sample_skeleton_pose` 中按 phase 索引帧 |
| ALVR/SteamVR 全身追踪 | 通过 OpenXR Body Tracking 扩展拿到全身节点 → 重定向到 G1 关节 |
| SMPL 参数化模型 | 解码 SMPL pose 参数 → 关节角度 |

数据替换后，仅需保证 `_sample_skeleton_pose(phase) → 关节角度张量` 这个接口不变，其他逻辑（根节点平移、关节写入）无需改动。

## 技术演进：从动画驱动到物理驱动

### 动画驱动方式（v1-v2，已废弃）

**实现方式**：
- 根节点固定（`fix_root_link = True`）
- 每帧通过 `write_root_state_to_sim` 直接修改根节点世界坐标
- 速度恒定（0.3 m/s），沿当前朝向移动
- 添加躯干竖向起伏（body bob）模拟重心变化

**优点**：
- ✅ 绝对稳定，不会摔倒
- ✅ 速度可控，轨迹精确

**缺点**：
- ❌ 脚不接触地面（"幽灵感"）
- ❌ 不是真实物理行走
- ❌ 无法应对地形变化
- ❌ 步幅与速度需要手动匹配

### 物理驱动方式（v3，当前版本）

**实现方式**：
- 根节点自由（`fix_root_link = False`）
- 启用重力（`disable_gravity = False`）
- 仅通过关节动画驱动，让物理引擎计算根节点运动
- 脚与地面产生真实接触力

**优点**：
- ✅ 真实物理行走
- ✅ 脚与地面接触，自然的重心转移
- ✅ 可以应对地形变化
- ✅ 步幅与速度自动匹配

**缺点**：
- ⚠️ 需要调整参数以保持稳定
- ⚠️ 可能在某些参数下摔倒

### 关键代码变化

**动画驱动（已移除）**：
```python
# 每帧强制移动根节点
fwd = self._forward_dir_from_quat(self._asset.data.root_quat_w)
self._walk_xy += fwd * (self.cfg.forward_speed * dt)
body_z = self._init_root_z + A_bob * torch.abs(torch.sin(self._phase))
target_pos = torch.cat([self._walk_xy, body_z], dim=-1)
self._asset.write_root_state_to_sim(root_state)
```

**物理驱动（当前）**：
```python

self._phase += 2.0 * math.pi * self.cfg.walk_frequency * dt
self._processed_actions = self._sample_skeleton_pose(self._phase)
```

## 调试与优化建议

### 如果机器人不稳定或摔倒

1. **降低步态频率**
   ```python
   walk_frequency: float = 0.6  # 从 0.8 降低到 0.6
   ```

2. **减小关节摆动幅度**
   ```python
   hip_pitch_amplitude: float = 0.20  # 从 0.25 降低
   knee_amplitude: float = 0.25       # 从 0.30 降低
   ```

3. **增加腰部侧倾（改善平衡）**
   ```python
   waist_roll_amplitude: float = 0.08  # 从 0.05 增加
   ```

4. **检查初始高度**
   ```python
   WALKER_G1_29DOF_CFG.init_state.pos = (-2.0, 0.0, 0.75)  # 确保 Z 轴高度合适
   ```

### 如果想要更快的行走速度

1. **提高步态频率**
   ```python
   walk_frequency: float = 1.0  # 增加到 1.0 Hz
   ```

2. **增加髋关节摆动幅度**
   ```python
   hip_pitch_amplitude: float = 0.30  # 增加步幅
   ```

### 如果想切换回动画驱动（用于演示）

1. **修改配置**
   ```python
   WALKER_G1_29DOF_CFG.spawn.articulation_props.fix_root_link = True
   WALKER_G1_29DOF_CFG.spawn.rigid_props.disable_gravity = True
   ```

2. **恢复根节点平移代码**（参考 commit e8fe4c15 之前的版本）

### 性能监控

在运行时观察以下指标：
- 根节点高度是否稳定
- 脚是否与地面保持接触
- 机器人是否保持直立姿态
- 行走轨迹是否平滑
