# 场景布局开关：流水线布局 ↔ 原布局

`Isaac-PickPlace-Locomanipulation-G1-*` 的 warehouse 场景有两套摆位，用一个开关
`ISAACLAB_TOTES_ON_CONVEYOR` 切换，配置全在
`source/isaaclab_tasks/isaaclab_tasks/manager_based/locomanipulation/pick_place/locomanipulation_g1_env_cfg.py`。

| | 流水线布局（默认，=1） | 原布局（=0） |
| --- | --- | --- |
| 两个塑料筐 | 缩小一半（scale 0.005 → 0.3×0.2×0.15 m），躺在流水线滚轮面上 | 原尺寸（scale 0.01 → 0.6×0.4×0.3 m），叠放在拖车顶面 |
| 筐坐标 | (-5.35, 16.4, 0.775) / (-5.89, 17.0, 0.775)，Y 是**起点** | 均 (-5.62, 18.75)，z=0.3794 / 0.6814（逐层 +2 mm 间隙） |
| pushcart_2 拖车 | (-5.4, 19.394) 的空车 | (-5.62, 18.75)，载着两筐顶在流水线入料端前 |
| 双机站位 | (-4.75 / -6.7, y=14.148) 流水线第二段两侧 | (-4.82 / -6.42, y=18.75) 推车两侧 |
| drive_totes 驱动事件 | 开（0.3 m/s 沿 -Y 送到工位停住） | 关 |

原布局的用途是**双机把筐从推车搬上流水线**：整组对齐流水线中线 x=-5.62，顶到入料端
（y=18.222）前方，机器人站定就能够到入料口。

## 怎么切

⚠️ **PowerShell 里 `set X=0` 不管用**——`set` 在 PowerShell 是 `Set-Variable` 的别名，
那是 cmd 的语法，环境变量根本没设上，而且不报错，看起来就像"代码没生效"。

```powershell
# PowerShell（win 遥操机常用）
$env:ISAACLAB_TOTES_ON_CONVEYOR = "0"
.\isaaclab.bat -p scripts\environments\teleoperation\teleop_se3_agent.py --xr --device cpu `
    --task Isaac-PickPlace-Locomanipulation-G1-Abs-v0 --teleop_device motion_controllers --enable_pinocchio
```

```bat
:: cmd
set ISAACLAB_TOTES_ON_CONVEYOR=0
```

变量只在当前终端窗口有效，换窗口要重设。想常驻就往 `scripts/gr00t_wbc/g1_udp_network.env`
里加一行 `ISAACLAB_TOTES_ON_CONVEYOR=0`——但那个文件两台机器共享且被 git 跟踪，改了会影响
另一端，一般还是用 `$env:` 临时设。

**启动日志会打印实际生效的布局和坐标**，一眼就能确认开关有没有吃到：

```
[locomanipulation_g1_env_cfg] 场景布局: 原布局（两筐原尺寸叠在拖车上） [ISAACLAB_TOTES_ON_CONVEYOR=0]
[locomanipulation_g1_env_cfg]   拖车/筐 x=-5.620 y=18.750 | robot_1 x=-4.820 robot_2 x=-6.420 y=18.750 | 流水线中线 x=-5.620 入料端 y=18.222
```

## 原布局的位置微调（不用改代码）

这几个量读进程环境变量优先，可以在 GUI/VR 里边看边试，定下来再改代码里的默认值：

| 环境变量 | 默认 | 作用 |
| --- | --- | --- |
| `ISAACLAB_CART_GROUP_Y` | 18.75 | 整组前后（推车 + 两筐 + 双机**联动**） |
| `ISAACLAB_CART_GROUP_X` | -5.62 | 整组左右 |
| `ISAACLAB_ROBOT_SIDE_OFFSET` | 0.80 | 双机相对推车的左右间距 |
| `ISAACLAB_ROBOT_1_X` / `ISAACLAB_ROBOT_2_X` | 中线 ±0.80 | 单独挪某一台机器人 |
| `ISAACLAB_ROBOT_WORKSTATION_Y` | = CART_GROUP_Y | 只挪机器人、不挪推车 |

```powershell
$env:ISAACLAB_TOTES_ON_CONVEYOR = "0"
$env:ISAACLAB_CART_GROUP_Y = "18.9"   # 整组后退 0.15 m
```

流水线布局的分项（`ISAACLAB_CONVEYOR_SPEED` / `_Y_STOP` / `_Y_RECYCLE` / `_Y_RESPAWN`、
`ISAACLAB_CONVEYOR_ENABLED`）走的是 `_cfg_*`，**只认 `g1_udp_network.env` 文件**，
进程环境变量对它们无效。

## 位置是怎么算出来的

场景实测几何：

* 流水线（背景 USD `/Root/ConveyorBelt`，ConveyorBelt_A08 ×3 段）滚轮带 x∈[-6.07,-5.17]，
  中线 **-5.62**；整体结构 x∈[-6.19,-5.04]；y 跨度 [10.19,18.22]，**入料端 y=18.222**；
  滚轮顶 z=0.772，侧护栏顶 1.169（所以筐只能从端部送入，越不过护栏）。
* 拖车 `SM_PushcartA_02` 经 scale (0.5,0.5,1.0) 后：X 宽 0.468、**Y 长 0.824（半长 0.412）**、
  顶面 z=0.3774——顶面高度与筐的落点 0.3774 一致，可反证资产和缩放没错。

于是原布局取 y=18.75：推车前缘 18.338，离带端留 0.116 m；机器人中心离带端 0.528 m，在臂展内。

双机间距取 **±0.80**，这是"机器人身体完全脱出流水线结构"的临界值：机器人半宽约 0.22，
robot_1 左缘正好落在 -5.04、robot_2 右缘 -6.20，贴着结构 x[-6.19,-5.04] 两侧擦过去。
早先用 0.65 时双机 x 区间与结构交叠约 0.15 m，遥操往 -Y 走两步就会撞流水线侧面。
代价是离筐中心 0.80 m（筐 X 半宽 0.30 → 手要侧伸约 0.5 m 够到筐边），接近 G1 侧向可达
上限，再拉开就该抓不到了。机器人与推车的侧向间隙则是 0.346 m（0.80 − 0.234 − 0.22）。

**两个下限**：整组 Y 不能小于 **18.634**（再小推车头插进流水线）；双机间距不宜小于
**0.80**（再小身体就与流水线结构交叠）也不宜超过约 0.90（手够不到筐）。

## 已知边界

1. 双机间距拉到 ±0.80 之后，Y（18.75 在流水线 y 跨度之外）和 X（身体擦过结构两侧边缘）
   两个方向都不与流水线干涉，遥操往 -Y 走也不会撞侧面。但这是**擦边**通过，若把
   `ISAACLAB_ROBOT_SIDE_OFFSET` 调回 0.65 一类的小值，撞侧面的隐患就回来了。
2. **机器人朝向没随开关变**：两套布局都是面对面（robot_1 朝 -X、robot_2 朝 +X）夹着中间的筐，
   所以原布局下"搬到流水线"是侧向动作。要改成正面朝流水线，得连 deploy 侧 relative root 的
   前进方向一起改。
3. **`conveyor_collider` 常驻**（托住筐的不可见 kinematic 碰撞板，y∈[10.19,18.22]）。原布局的
   物体都在 y≈18.75，在板范围外，留着没有副作用。
4. **背景 USD 的改动不随开关回退**：第二段两侧的打包桌 / 分拣料箱平移 +0.6 m 和左右镜像已经
   烘进 `warehouse-simple6_v48.usd`，代码开关碰不到。要退得换回同目录的
   `.bak-workstation-shift-20260721` / `.bak-symmetric-20260721`。这些件都在 y≈15 附近，
   不影响原布局。
5. ⚠️ **纸箱推车组（x=-6.8, y=19.394）没跟着挪**。robot_2 拉到 -6.42 后与它 **X 方向已交叠
   0.074 m**，全靠 Y 方向 **0.082 m** 的间隙错开——机器人往 +Y 后退一步就会顶到纸箱推车。
   要给 robot_2 留活动余量，得把纸箱推车组也往 +Y 或 -X 挪（本次未动）。
